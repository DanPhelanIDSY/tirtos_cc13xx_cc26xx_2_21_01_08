  • CC13xxWare RF source code [RF_SRC]
  • CC13xxWare RF documentation [RF_DOC]

cc13xxware_2_04_02_17240

Release date: June 22, 2016

New features

  • [RF_SRC] Header files: Added RF_MODE_MULTIPLE define to rf_mailbox.h for use on CC1350 in dualmode.

Improvements

  • [RF_SRC] Patches: Updated the GENFSK RFE patch with improved internal PA ramping.
  • [RF_SRC] Patches: rf_overrides_ble file is deprecated. Override lists can be found in SmartRF Studio 2.4.0 or later from now on.
  • [RF_SRC] Patches: General improvement on BLE RFE patch. Related to HPOSC mode.
  • [RF_SRC] Header files: Removed unused LRM MCE patch
  • [RF_SRC] Patches: Reduced size of GENFSK/OOK and LRM CPE patches.
  • [RF_SRC] Header files: Removed unused fields in CMD_SET_TX_POWER.
  • [RF_SRC] Header files: Removed unused fields in CMD_FS.

  • [RF_SRC] Patches: Added new GENFSK RFE patch to improve internal PA ramping.

Bug fixes

  • [Setup] Adjusted voltage difference between digital supply and VDD BOD during STANDBY.
  • [Setup] Adjusted battery monitor low limit when exiting STANBY.
  • [RF_SRC] Patches: New BLE and BLE PRIV 1.2 CPE patch fixing issue with faulty output.

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.

cc13xxware_2_04_01_17076

Release date: May 9, 2016

New features

  • [RF_SRC] RFC: Added new function RFCAdi3VcoLdoVoltageMode() for setting VCOLDO reference mode.
  • [RF_SRC] RFC: Added GENFSK RFE patch to improve PA ramping.
  • [RF_SRC] RFC: Added new functions to check RTrim override value against limit in flash.
  • [RF_SRC] Header files: Added new field analogCfgMode in setup commands to allow for faster startup.
  • [RF_SRC] RFC: Added new function RFCCPEPatchReset() in order to reset patched CPE ram.
  • [DRV_SRC] ChipInfo: Added new functions:
    • ChipInfo_HwRevisionIs_GTEQ_2_1()
    • ChipInfo_GetUserId()
    • ChipInfo_GetChipType()
  • [DRV_SRC] Crypto: Added new functions to support the AES_CBC mode:
    • CRYPTOAesCbc()
    • CRYPTOAesCbcStatus()
    • CRYPTOAesCbcFinish()
  • [DRV_SRC] CCFG: Added check in boot function trimDevice() to support external HF clock (e.g. TCXO). This is done by setting XOSC_HF in bypass mode if CCFG setting DIS_TCXO=0.

Improvements

  • [RF_SRC] Patches: New BLE RFE patch to improve TX startup sequence.
  • [RF_SRC] RFC:Improved RF header file comments.
  • [RF_SRC] RFC:Removed unused bit fields from commands to simplify structures.
  • [RF_SRC] RFC:Merged txPower field from three separate fields down to one 16 bit field.
  • [DRV_LIB] CCS: Changed compile options to optimize for size (opt_level=4 opt_for_speed=0).
  • [DRV_DOC] General: Module introduction is available in the driverlib folder (<Module>_doc.h).

Bug fixes

  • [RF_SRC] Patches: All patches updated with fix for CMD_GET_RSSI. The command would fail without error indication if issued while the radio was powering down.
  • [DRV_SRC] Crypto: Corrected possible timing race in crypto driver. Changes done in the following functions:
    • CRYPTOAesEcb()
    • CRYPTOCcmAuthEncrypt()
    • CRYPTOCcmInvAuthDecrypt()
  • [DRV_SRC] Startup: Does not enable DC/DC when device CCFG is configured for external regulator mode.

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.

cc13xxware_2_04_00

Release date: March 03, 2016

New features

  • [RF_SRC] Patches: Added BLE 4.2 link layer privacy patch.
  • [DRV_SRC] OSC: Added test/debug functions to read out crystal amplitude. Added the following functions:
    • OSCHF_DebugGetCrystalAmplitude()
    • OSCHF_DebugGetExpectedAverageCrystalAmplitude()

Improvements

  • [RF_SRC] Patches: Updated BLE 4.0/4.1 patch.
  • [DRV_SRC] OSC: The OSC interface is enabled at startup and must be kept on in order to let other sub modules work properly. The functions OSCInterfaceEnable() and OSCInterfaceDisable() are therefore converted to empty functions and marked as deprecated.
  • [DRV_SRC] sys_ctrl: Removed functions (can be harmful to use):
    • SysCtrlStandby()
    • SysCtrlPowerdown()
    • SysCtrlShutdown()

Bug fixes

  • [DRV_SRC] aux_timer: Corrected problem with aux_timer configuration resulting in always configuring both AUX_TIMER_0 and AUX_TIMER_1. This applies to the following functions (access to these functions in ROM is now disabled):
    • AUXTimerConfigure()
    • AUXTimerStart()
    • AUXTimerStop()
    • AUXTimerPrescaleSet()
    • AUXTimerPrescaleGet()
  • [DRV_SRC] aux_timer: Corrected defines AUX_TIMER_PRESCALE_DIV_* used as input parameters to the following functions:
    • AUXTimerPrescaleSet()
    • AUXTimerPrescaleGet()

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.

cc13xxware_2_01_00_16101

Release date: September 28, 2015

New features

  • [DRV_LIB] GCC: Added GCC prebuilt library (Now supporting: IAR, CCS, Keil/ARM and GCC).
  • [DRV_SRC] OSC: Added functions to enable and disable clock loss reset functionality (New functions: OSCClockLossEventEnable() and OSCClockLossEventDisable()). Note that for TI-RTOS users, clock loss reset is now enabled automatically when all clocks are stable after power up/reset.
  • [DRV_SRC] VIMS: Renamed function VIMSModeSetBlocking() to VIMSModeSafeSet() and changed the function's API. Added a new parameter ('blocking') to be set if the function shall wait until the change has completed.
  • [DRV_SRC] AON: Changed API and renamed input parameters for functions AONIOCDriveStrengthGet() and AONIOCDriveStrengthSet(). See documentation for usage.
  • [DRV_SRC] General: Cleaned up defines. Applies to the following defines:
    • Removed XoscInHighPowerMode (replaced by XOSC_IN_HIGH_POWER_MODE)
    • Removed XoscInLowPowerMode (replaced by XOSC_IN_LOW_POWER_MODE)
    • Removed AUX_TDC_SATCFG_LIMIT_ROVF (replaced by AUX_TDC_SATCFG_LIMIT_R24)
    • Removed IOC_CURRENT_16MA (not valid)

Improvements

  • [DRV_DOC] General: Documentation for all IntClear() functions updated to clarify how to safely clear an event in an Interrupt Service Routine (ISR).

Bug fixes

  • [DRV_SRC] SysCtrl: Corrected typo in the initial recharge period calculation when entering standby, lowering power consumption in some scenarios.
  • [DRV_SRC] AON: Define AON_EVENT_DIO0 as unconnected if package type is 7x7 mm (since DIO0 does not exist on the CC13xx 7x7 package).

Known Issues

cc13xxware_2_00_04_16084

Release date: September 22, 2015

New features

  • [DRV_SRC] CCFG: Added support for forcing "high VDDR" voltage trim through CCFG configuration (by setting define CCFG_FORCE_VDDR_HH=1). This is done in order to enable boost mode.
  • [RF_SRC] Radio: Removed support for the previously added Command and Packet Engine (CPE) command, CMD_TX_POWER_BOOST, as modification of VDDR voltage trim is no longer supported as a radio controlled function. This is instead controlled by the system processor as part of system configuration from CCFG at boot.
  • [DRV_SRC] SysCtrl: Updated initial recharge period calculation when entering standby to support the "high VDDR" trim option.

Improvements

  • [RF_SRC] Radio: Updated RF Core patch files for Command and Packet Engine (CPE) to include updated PLL loop filter settings for the RF synthesizer.
  • [RF_SRC] Radio: Updated RF Core patch files for Command and Packet Engine (CPE) to include support for dynamic adjustment of DC/DC voltage regulator configuration so that the recommended settings will be used when in Rx or Tx mode.
  • [RF_SRC] Long range mode: Added a separate RF Core patch file for Command and Packet Engine (CPE) to be used for the proprietary Long range mode (LRM) PHY modes.

Bug fixes

  • [DRV_SRC] SysCtrl: Corrected typo in the initial recharge period calculation when entering standby (corrected minor bug in the temperature compensation calculation).

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.

cc13xxware_2_00_03_15980

Release date: August 4, 2015

IMPORTANT NOTICE
This is a pre-release version of software for the CC1310, and it should not be used in production of CC1310 devices.
The CC1310 production release software is in its final stages of development and verification, and it should be expected that radio-specific timings and parameters will change in that release.

New features

  • [RF_SRC] GenFSK: Added new GenFSK command - CMD_TX_POWER_BOOST.
  • [DRV_SRC] Event: Added functions to set and clear software events (EventSwEventSet, EventSwEventClear and EventSwEventGet).
  • [DRV_SRC] AON_RTC: Added function AONRTCCurrent64BitValueGet(), returning the SEC and SUB_SEC registers as one 64 bits result.
  • [DRV_SRC] CCFGRead: Added new module with a set of functions returning most of the CCFG bit fields.
  • [DRV_SRC] CCFG: Added CCFG configuration option to use CACHE RAM as general purpose RAM (GPRAM) - (Choosing between having cache or having 8 K-bytes of extra RAM).
  • [DRV_SRC] CCFG: Added ability to change the CCFG struct location. This is a special functionality needed only if executing entirely from RAM. Note that the prebuilt library is built with default CCFG location and will not work properly if CCFG location is changed.

Improvements

  • [RF_SRC] General: TX performance enhancements.
  • [DRV_SRC] AON_RTC: Changed function AONRTCCurrentCompareValueGet() to avoid disabling interrupts.
  • [DRV_SRC] General: Code size optimization of several modules.
  • [RF_DOC] General: Integrated "RF" documentation into the existing documentation structure.

Bug fixes

  • [DRV_SRC] OSC: XOSC startup time calculation done by function OSCHF_GetStartupTime() returned maximum startup time too early.
  • [DRV_SRC] CPU: Corrected compiler error when compiled with GCC.

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.

cc13xxware_2_00_02_15830

Release date: June 11, 2015

New features

  • [DRV_SRC] ROM: SW-based crypto functions in ROM are now available through DriverLib (see rom_crypto.h).
  • [RF_SRC] Radio: Restructuring radio library: Moving rflib files to driverlib (prefixed with "rf_"), added rf_patches catalog (for RF patche files).

Improvements

  • [DRV_SRC] General: Minor code size reduction in several module APIs by using bit banding when possible and replacing base address argument with a constant in single-instance modules.
  • [DRV_SRC] General: Compiler specific defines are renamed to use compiler's own defines. gcc --> __GNUC__ , rvmdk --> __CC_ARM . Compiler defines 'codered' and 'sourcerygxx' are deleted.
  • [DRV_SRC] setup: Some TI-RTOS power startup configuration is moved to trimDevice() to run pre-RTOS.
  • [DRV_SRC] setup: Setup API now has a setup.h file which can be included instead of declaring trimDevice() as an extern explicitly. The setup API is added to the DriverLib HTML documentation.
  • [DRV_SRC] AON_RTC: AONRTCStatus is split into AONRTCActive and AONRTCChannelActive for faster and more compact code.
  • [DRV_SRC] AUX_TDC: Defines in aux_tdc.h now refer to defines from hw_aux_tdc.h to avoid magic numbers and make a stronger connection to register descriptions. All defines for start and stop conditions are renamed to match enumerations in register description.
  • [DRV_SRC] AUX_TIMER: Defines in aux_timer.h now refer to defines from hw_aux_timer.h to avoid magic numbers and make a stronger connection to register descriptions.
  • [DRV_SRC] AUX_WUC: Defines in aux_wuc.h now refer to defines from hw_aux_wuc.h to avoid magic numbers and make a stronger connection to register descriptions.
  • [DRV_SRC] AON_EVENT: AON Event defines for event selection are renamed to better match register description enumerations.
  • [DRV_SRC] CCFG: Cleanup in the customer configuration description (CCFG/ccfg.c)
  • [DRV_SRC] hw_cpu: Removed obsolete file: hw_cpu_cc26xx.h

Bug fixes

  • [DRV_SRC] General: Crypto and Flash APIs are updated to fix GCC compiler issue.
  • [DRV_SRC] Event: Defines in event.h used for EventRegister() are deleted and instead the documentation describes how to select defines from hw_event.h and how to find valid pairs of event sources and event subscribers.
  • [DRV_SRC] SysCtrl: SysCtrlSetRechargeBeforePowerDown() now uses C basic type arguments instead of enumerated types. Defines for valid arguments are added and documented.
  • [DRV_SRC] SysCtrl: SysCtrlSetRechargeBeforePowerDown() is updated to configure recharge configuration slightly different, reducing maximum recharge period.
  • [DRV_SRC] AUX_ADC: External ADC reference is not supported thus removed from AUX_ADC API.
  • [DRV_SRC] AUX_ADC: AUXADCReenableInputScaling() is removed from AUX_ADC API and safe usage of AUXADCDisableInputScaling() documented.
  • [DRV_SRC] AUX_ADC: Configuration of AUX ADC is changed into two steps so it won't be enabled until after configuration is done.
  • [DRV_SRC] PRCM: PRCMWdtResetStatus() is removed since a WDT reset is no longer detectable after setting WR_TO_PINRESET bit.
  • [DRV_SRC] Timer: TimerTriggerControl() is removed because it access functionality which is not supported.

Known Issues

  • None
  • For an updated list of known issues (including issues found after the release), see CC13xxWare on the Texas Instruments wiki.
