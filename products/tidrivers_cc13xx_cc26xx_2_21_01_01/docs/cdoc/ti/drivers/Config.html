VALUES
LibType_Instrumented — The library supplied is pre-built with logging and assertions enabled. Diags_USER1 mask logs basic information Diags_USER2 mask logs more detailed information
LibType_NonInstrumented — The library supplied is pre-built with logging and assertions disabled.
DETAILS

TI Drivers library type

XDCscript usage meta-domain
Config.libType = Config.LibType Config.LibType_NonInstrumented;
 
DETAILS
The driver libraries are provided to link with your application. Two forms of this library are provided with the TI Drivers product. This configuration parameter allows you to select the form of the driver libraries to use.
The default value of libType is LibType_NonInstrumented. For a complete list of options and what they offer see LibType.
 
config Config.rfDriverMode  // module-wide

TI-Drivers RF Driver mode

XDCscript usage meta-domain
Config.rfDriverMode = Config.RFDriverMode Config.RF_SingleMode;
 
DETAILS
The RF driver supports single & multi modes of operation. In single-mode only a single client/protocol can interact with the radio, meaning if another protocol is required, the RF driver must be closed & re-opened.
In multi-mode more than one client/protocol can interact with the radio simultaneously.
The default value of rfDriverMode is: RF_SingleMode. For a complete list of options and what they offer see RFDriverMode.
