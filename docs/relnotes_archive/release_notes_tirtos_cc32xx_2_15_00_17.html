Introduction

TI-RTOS delivers components that enable engineers to develop applications on Texas Instruments micro-controller devices. The product is comprised of multiple software components and examples of how to use these components together. TI-RTOS gives developers a one-stop RTOS solution for developing applications for TI embedded microcontrollers. It provides an OS kernel, communications support, drivers, and more. It is tightly integrated with the Code Composer Studio development environment. In addition, examples are provided to demonstrate the use of each functional area and each supported device and as a starting point for your own projects.

What’s New

To see what has changed in the TI-RTOS Drivers, see the separate TI-Drivers release notes in [as installed]/products/tidrivers_M_mm_pp_bb.

  • (TIRTOS-71) MacOS support. This product can be used with Code Composer Studio on MacOS.
  • (TIRTOS-118) The former TI-RTOS for SimpleLink has been split into two separate products: TI-RTOS for CC32xx and TI-RTOS for CC13xx and CC26xx.
  • (TIDRIVERS-14) Driver modules (xdc/xs/xml) files have been removed from TI-Drivers. The TIRTOS module has been updated to provide warnings if the useDRIVER parameters are present in *.cfg.
  • ROV support has been removed from all of the TI-RTOS Drivers.
  • The host driver for the CC3200 WiFi device is now included in the product along with an WiFi example for the CC3200.

Upgrade and Compatibility Information

The following is a summary of known compatibility issues from previous releases:

  • See this wiki page for help with migration from a prior release of TI-RTOS to TI-RTOS 2.15: http://processors.wiki.ti.com/index.php/TI-RTOS_Migration_2_15
  • Short summary of compatibility breaks from prior releases. See the above wiki page for more details.
    • When using IAR workbench, the generated ‘configPkg’ directory has moved in some project environments. Some IAR projects may have to be modified.
    • The ti/drivers source code and libraries have been moved into a separately versioned ‘tidrivers’ component. The source for the drivers can be found in [as_installed]/products/tidrivers_M_mm_pp_bb.
    • The Power module has been moved from the kernel component (ti/sysbios/family/arm/cc26xx/Power.h) to the drivers component (ti/drivers/Power.h). Any code that references Power.h will have to be updated. There were also some minor API and #define changes.
    • The board files for the CC13xx and CC26xx boards have been renamed and the directory structue in ‘ti/boards’ has been modified.
    • The GPIO driver callback function signature has changed.
    • The configuration modules and ROV support for UART, GPIO, etc. have been removed. Configuration scripts (.cfg) that have ‘useModule’ for these modules will have to be updated to remove the ‘useModule’ statements.
    • The source and libraries for FatFS have been moved from the SYS/BIOS product to the TI-Drivers product.
    • 2 new configuration modules have been added – ‘ti.drivers.Config’ and ‘ti.mw.Config’. These should be used in the project .cfg file to pull in the driver and middleware libraries. The ‘ti.tirtos.TIRTOS’ module is provided for backwards compatibility.
    • The CC13xx/CC26xx LCD driver has been moved from ‘ti/drivers/lcd’ to ‘ti/mw/lcd’ since this is middleware and not really a driver.
    • See the above wiki page for more additional migration issues.

Versioning

All releases have 4 digits (M.mm.pp.bb). This includes GA and engineering releases. Engineering releases are denoted with an _eng suffix (e.g., 2.00.00.15_eng)

This product’s version follows a version format, M.mm.pp.bb, where M is a single digit Major number, mm is 2 digit minor number, pp is a 2 digit patch number, and b is an unrestricted set of digits used as an incrementing build counter.

To support multiple side-by-side installations of the product, the product version is encoded in the top level directory (e.g. tirtos_2_00_00_16).

Subsequent releases of patch upgrades will be identified by the patch number, ex. TI-RTOS 1.00.01.68 with directory tirtos_1_00_01_68. Typically, these patches only include critical bug fixes.
